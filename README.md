# Huntsman

Huntsman is a standalone Dalamud plugin for drop-material hunting. It searches local game data, builds a manual hunt list, routes to monsters, and hands off combat once the target is reached.

## Features

- Local LuminaSupplemental mob-drop data with embedded route-data fixes.
- Droppable item search with quantity selection.
- Manual hunt lists that merge duplicate item requests.
- Route planning to monster spawn clusters.
- Lifestream teleport support with a configured `/tp` fallback.
- vnavmesh movement between route points and visible targets.
- Auto-mount and auto-dismount behavior.
- RotationSolverReborn or WrathCombo combat driver readiness checks.
- Diagnostics for data, navigation, mount, and combat state.

Huntsman is fully standalone and does not require external crafting or gathering automation plugins.

## Commands

- `/huntsman`: open the main window.
- `/huntsman open`: open the main window.
- `/huntsman status`: print data, navigation, mount, and combat status.
- `/huntsman stop`: stop the active hunt and navigation.

## Build

Open `Huntsman.slnx` with a Dalamud API 15 development environment. The plugin project targets `net10.0-windows` and expects Dalamud dev assemblies under `%APPDATA%\XIVLauncher\addon\Hooks\dev\`.

```powershell
dotnet restore Huntsman.slnx
dotnet build Huntsman/Huntsman.csproj --configuration Release
```

## Dalamud Custom Repository

Use this URL in Dalamud's Custom Plugin Repositories list:

```text
https://raw.githubusercontent.com/Ankhito/AnkhitoPlugins/main/pluginmaster.json
```

The repository manifest should point to the latest GitHub release asset named `Huntsman-latest.zip`, generated by DalamudPackager.
